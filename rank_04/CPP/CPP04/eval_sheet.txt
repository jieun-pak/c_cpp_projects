## Guidelines

* Only grade the work that was turned in the **Git repository**.
* Double-check that the repository belongs to the correct student(s).
* Ensure `git clone` is used in an empty folder.
* Check carefully that no malicious aliases were used.
* Review any scripts used for testing or automation.
* If you haven’t completed the assignment yourself, **read the entire subject first**.
* Use available flags to report:

  * Empty repository

## Prerequisites

* Code must compile with:

  ```bash
  c++ -Wall -Wextra -Werror
  ```
* Must follow **C++98 standard**
* C++11 or later features are **not allowed**

### Do **NOT** grade if:

* A function is implemented in a header file (except templates)
* Makefile does not use required flags or compiler

### Flag **Forbidden Function** if:

* Use of C functions (`malloc`, `printf`, `free`, etc.)
* Unauthorized functions
* `using namespace`
* `friend` keyword
* External libraries or non-C++98 features

## Ex00: Polymorphism

### Requirements

* A `main` function with sufficient tests
  → **No main = no grade**
* All non-interface classes must follow **orthodox canonical form**

### Animal Class

* Attribute:

  * `std::string type`
* Must be instantiable and usable

### Inheritance

* Classes inheriting from `Animal`:

  * `Cat`
  * `Dog`
* Constructor & destructor outputs must be clear
* Student should explain constructor/destructor order

### Polymorphism

* `makeSound()` must be **virtual**

  ```cpp
  virtual void makeSound() const;
  ```
* Correct method must be called via base pointer/reference

### WrongAnimal Test

* `WrongAnimal` and `WrongCat` **without virtual**
* `WrongCat` should only call its own `makeSound()` when used directly

## Ex01: I Do Not Want to Set the World on Fire

### Concrete Animal

* New class: `Brain`
* `Cat` and `Dog` contain a **private Brain**
* `Brain` must **NOT** be inside `Animal`
* `Brain` has output on creation & destruction

### Deep Copy

* Copy must be **deep**

* Example test:

  ```cpp
  Dog basic;
  {
      Dog tmp = basic;
  }
  ```

* Shallow copy causes double delete → ❌

* Copy constructor must also perform deep copy

* Proper canonical form is mandatory

### Destruction Chaining

* Destructors must be **virtual**
* Student must explain what happens without `virtual`

## Ex02: Abstract Class

### Abstract Animal

* `Animal::makeSound` is **pure virtual**

  ```cpp
  virtual void makeSound() const = 0;
  ```
* Must NOT be instantiable

  ```cpp
  Animal test; // compilation error expected
  ```

### Concrete Animals

* `Cat` and `Dog` still exist
* Behavior identical to previous exercises

## Ex03: Interface and Recap

### Interfaces

* `ICharacter`
* `IMateriaSource`
* Must match subject exactly

### MateriaSource

* Implements `IMateriaSource`
* Member functions behave correctly

### Concrete Materia

* `Ice` and `Cure` inherit from `AMateria`
* `clone()` implemented correctly
* Correct output messages

### AMateria

* Abstract class
* `clone()` is pure virtual
* Virtual destructor present

  ```cpp
  virtual ~AMateria();
  ```
* Protected `std::string type`

### Character

* Implements `ICharacter`
* Inventory of **max 4 Materias**
* Member functions follow subject
* Copy & assignment are **deep copy**
